---
title: Geyser 主机提供商模组服务器设置
description: 了解如何在您的主机提供商的 Fabric 或 NeoForge 服务器上设置 Geyser。
---

import { Versions } from '@site/src/components/Versions'

:::caution
Geyser-Fabric 和 Geyser-NeoForge 仅在 <Versions platform="java"/> 服务器上运行。
有关更多详细信息，请参阅[此页面](/wiki/geyser/supported-versions#fabric-neoforge-servers)。
:::

:::info
Geyser 仅适用于**服务器端**模组。需要客户端安装的模组将无法工作！
:::

1. 在上面的下拉菜单中选择您的服务器主机提供商。
2. 从[下载页面](/download)下载模组。
3. 将 Geyser 模组 jar 文件放入 `mods` 文件夹，然后重启服务器。**对于 Fabric，您还需要 [FabricAPI](https://modrinth.com/mod/fabric-api)。**
4. 打开您的 Geyser 配置文件，位于 `/config/Geyser-Fabric/config.yml`（或 `/config/Geyser-NeoForge/config.yml`），找到以下内容：

    ```yaml
    bedrock: 
        # 将监听连接的 IP 地址。
        # 通常，只有在您想限制哪些 IP 可以连接到您的服务器时才应取消注释并更改此项。
        #address: 0.0.0.0 

        # 将监听连接的端口。这是基岩版玩家用来连接到您服务器的端口。
        port: 19132 

        # 某些主机服务每次启动服务器时都会更改您的 Java 端口，并要求对 Bedrock 使用相同的端口。
        # 此选项使 Bedrock 端口在每次启动服务器时都与 Java 端口相同。
        # 此选项仅适用于插件版本。
        clone-remote-port: false 
    ``` 
    关键部分是端口。这是基岩版玩家用来连接到您服务器的端口。
    按照提供商选择器中的说明操作 - 更改端口，如果需要，更改 `clone-remote-port` 和 `address`。
    如果启用了 `clone-remote-port`，基岩端口将被 Java 端口覆盖！
    **重要**：其他依赖 UDP 端口的服务/模组，例如语音聊天或查询，不能与 Geyser 共享端口。

5. 通过在控制台中运行 `geyser connectiontest <ip> <port>` 来验证是否可以进行连接。

:::info
要允许基岩版玩家无需登录付费 Java 版账户即可加入您的服务器，您可以使用 <a href="/wiki/floodgate/setup/">Floodgate</a>。
:::
